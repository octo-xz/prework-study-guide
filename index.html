HTML

  • !DOCTYPE - a declaration that tells the browser what type of document it should expect when its opened
  • The opening tag represents the start of a HTML document. Inside the tag is the attribute lang="" which informs the browser what language the web page will be in
  • The 'head' element contains metadata for a web application. It includes tags such as:
    • style - element defines css styles associated with a website
    • base - element that defines the base URL for a web page
    • link - an element that connects an external resource to the HTML doc
    • script - an element thtat embeds client-side scripts into the HTML doc
  • The title element defines the title for the webpage. It is a required and extremely important element for SEO (Search Engine Optimization.) It is contained in the head element.
  • The viewport element sets the default view for a webpage and makes for a better viewing experience on multiple different devices. It is also contained in the head element.
  • Meta elements define metadata about a HTML document. It is also inside the head element. Some meta attributes include:
    • charset - specifies character encoding for a HTML document
    • content - specifies the value associated with the http-equiv or name attributes
    • http-equiv - provides an HTTP header for the information/value of a content attribute
  • The body element defines the body of HTML document. It contains content that is visible to the user and all conent to be viewed by a user must be in the body element. These notes are currently in the body element.
  • Headers h1 to h6 are representative of the level of heading for a textblock. The headers for each section of note at h2. Headers must always be put in order from h1 to h6 and there should only be one h1 for each web page.
  • The p represents a paragraph or block of text in a HTML document.
  • The elements ul ol and li are used to make different types of lists. The ul is used for unordered lists and the items are listed with the li tag. These notes are an example of an unordered list. The ol tag stand for ordered list and also uses the tag li to list items.
  • The alt attribute can be used to describe an image on a web page for screen readers and greater SEO.
  • The img element is used to display images on a web page. It is used by containing information about images using the attribute src. The src attribute holds its importance by defining the location of an image to be displayed.
  • The br is a tag that contains an empty space or a line break between two blocks of text.
  • The a element stands for anchor and it is used in combination with the href attribute to create links to the same or different web pages. It is used in the format a = hrek"link would be here" with the name for the link here between open and closing brackets.
  • A section is what we are in right now! More specfically it is a semantic element the defines a section of a document that contains thematic content. Our current "theme" is HTML notes!
  • The id attribute is used in a HTML document to identify and select certain elements. This becomes important when CSS is added to a web page.
  • The footer element contains information at the bottom of a web page. This can include copyright, author, contact information, a sitemap and navigation.
  • Other mentions:
    • Semantic HTML elements are elements that make it easier to read a HTML document. They are important because the ease of reading is necessary for the collaborative nature of coding. Using semantic elements also impacts how accessible a site is to the broadest variety of users accessing it. It is considered best practice to use semantic elements wherever possible.
    • Elements vs tags. An element includes everthing from the opening and closing tag whereas a tag only refers to the information contained between angle brackets. The two terms are still often used interchangeably.
    • Element descriptors. Elements that are nested inside other elements (an example being meta elements inside the head tag) are called child elements while the element they are nested in is called a parent element.

CSS

  • Three types of CSS style sheets:
    • Inline CSS - put directly into an HTML file. Customizes elements by each individual tag and can only be edited individually. Is used by using the style tag.
    • Internal CSS - put directly in an HTML file in the head element. Also edits each element individually. Changes made to a specfic tag make changes to all tags on that name. For example if the h1 tag was assigned the color red, every h1 tag would become red.
    • External CSS - is put in a separate sheet that is linked to the HTML file using the link element; link rel="stylesheet" href=" name of the style file is usually just style.css" this method is the most common when to use CSS styling elements which follows the separation of concerns model. there are a few main components to external CSS:
      • selector - chooses which element are being selected for styling. * selector is used to apply a change to the entire page.
      • declaration - a statement with two parts; property: value and if there are multiple declarations for one selector all of the declarations are referred to as a declaration block. Property is what we want to apply to the value. An example is color: blue. We want to assign the value 'blue' to the property 'color'
    • CSS Box Model - every element within an HTML document is contained within a box. Each box has margins which are not visible on the outside of an element, a border which goes around the element, padding which is between the content(which is the final box) and the border. We can utilize and customize these boxes using CSS
    • Class attributes can be assigned in the HTML file to certain elements and can be set as a selector which is then customized with declarations in CSS

Git

  • Version Control: Allows tracking and mangement of changes made to a codebase. Whenever an version is saved with version control it is assigned a SHA or hash identifier which allows a record of changes made to be kept and also be accessible if reverting to an older version of the code is necessary. Only the first 7 numbers need to be typed out for a devloper to access a previous version which saves time that would be spent typing out the entire thing.
  • Git revert can be used to revert to a previous version of code. Can keep track locally.
  • Git - a version control system that allows managing and keeping track of each version of code of a project possible. Git is access in the command line through Git Bash. Git commands always start with 'git'. Using git commands make creating a repository, adding code to the repository and view different versions of code possible with each version of code stored in a unique commit.
  • GitHub - a cloud-based hosting service that allows for keeping track of repositories that are stored remotely. It also allows for collaboration to take place which is fundamental to how most coding works in practice. There are other git-based hosting repositories as well including BitBucket and SourceForge. The advantage of cloud based models is allowign multiple people to work to make separate changes to a program at the same time without conflicting issues.
  • Repository - A project on github is called a repository, this is where all the work that is made on the code including past versions is stored. More specfically, what is stored in a repo:
    • files that contain code
    • files that contain other information about the code(ex. README file)
    • the version history for all the files in the repo
    • a licensing agreement
  • Branch - A workspace where changes that don't affect the live site can be made. Making a new branch to store new work made to the code is best practice and should be the start to any new changes made.
  • Markdown(.md) - This is how content files are formatted on Github; it is a way to write for the web using plain text and it converts easily to HTML. Markdown files are marked with the .md file extension.
  • Commit changes - Saving a new version of a file that is being worked on. Whenever a file is added, edited or deleted a commit is made. It adds changes to the branch currently being worked in and is part of the pull request process.

JavaScript

  • Basic JavaScript features include:
    • Strings:
    • Variable: A container that stores a value; can hold values of different data types; if declared with keyword 'let'
    • Arrays: Is a way of listing multiple value in one container. Values that can be listed include strings, numbers, and booleans
    • Functions: A way to reuse blocks of code. Functions have two parts, the declaration and the call. The declaration defines the paramenters in which a function is called and what it will do and the call executes the function.
    • for Loops: One of the most basic types of loop. A loop runs through a block of code until the conditions for running are no longer met.
    • if Statements: A type of conditional statement. It compares whether or not a statement returns true. If the statement is true defined paramenters are executed, if it returns false other parameters are executed.
  • Data Types: strings, number, boolean, array, object
  • Control Flow - The order in which code is executed. This can be circumvented with functions or loops. Code in JavaScript is executed from top to bottom so the order in which code is written is very important.
  • OOP - Object Oriented Programming; JavaScript is an OOP language
  • Object - Everything in JavaScript is an object that can be stored in a variable.
  • Operators:
    • + Addition and concatenation of strings(joining two separate strings together)
    • - Subtraction
    • / Division
    • * Multiplication
    • % Modulo/Remaindefr
